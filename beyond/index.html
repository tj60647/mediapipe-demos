Beyond Face and Hands

MediaPipe offers many more sensing capabilities beyond the face and hand demos in this project. This page introduces five additional tasks using the same comparison questions so you can evaluate which one fits a project idea before writing any code.

← Back to Gallery

How to use this page

Before choosing a task for a project, use MediaPipe Studio to try it live in your browser with no setup. Then use the questions below as a frame for evaluating whether the task fits your idea.

  • What does this task detect?
  • What kind of input does it need?
  • What kind of output does it return?
  • What kind of interaction becomes possible?
  • What are the likely failure cases?

Tasks

Object Detection
Object Detection
Detect and label everyday objects in a camera frame
Detects
Up to 80 object categories (people, chairs, bottles, phones, cups, animals, vehicles, and more)
Input
Webcam feed, image, or video file
Output
Bounding box coordinates and label for each detected object, with a confidence score
Interaction
Each object becomes a trigger, controller, or character; the scene changes based on what is placed in front of the camera
Failure cases
Overlapping objects, low contrast, partially visible items, objects not in the 80-category set
Project prompt
Domestic Object Orchestra
Place everyday objects in front of the camera and assign each detected object a sound, animation, or role in a scene. Build a system where common objects become controllers for a playful composition, game, or performance. Make the system reveal something strange or funny about ordinary things.
Try in Studio
Gesture Recognition
Hand Gesture Recognition
Classify recognised hand gestures from a webcam feed
Detects
Seven built-in gestures: None, Closed Fist, Open Palm, Pointing Up, Thumbs Down, Thumbs Up, Victory, ILoveYou
Input
Webcam feed, image, or video file
Output
Gesture label and confidence score per hand, plus the underlying 21 hand landmarks
Interaction
Gestures become discrete commands, scene switches, or ritual triggers; each gesture can unlock a different state or effect
Failure cases
Ambiguous hand poses, gestures outside the seven built-in categories, fast motion blur, poor lighting
Project prompt
Secret Handshake Operating System
Create an interface that only responds to a set of deliberate gestures. Recognised gestures trigger different interface states, hidden messages, or scene changes. Treat gestures as spells, rituals, commands, or social signals rather than simple buttons.
Try in Studio
Pose Tracking
Pose / Holistic Tracking
Track the full body using 33 pose landmarks
Detects
33 body landmarks including shoulders, elbows, wrists, hips, knees, ankles, and facial anchor points
Input
Webcam feed, image, or video file; the full body should be visible for best results
Output
Normalised 3D landmark positions (x, y, z) for each of the 33 joints
Interaction
Posture, balance, arm angles, and movement quality all become inputs; the body can control environments, characters, or sound
Failure cases
Partial body in frame, occlusion by furniture, low-contrast clothing, very fast movement
Project prompt
Full-Body Puppet Weather System
Use whole-body tracking to build an environment that reacts to posture, balance, or movement quality. Body position changes the environment — wind, rain, particles, or lighting. Think of the body as a weather controller, stage effect controller, or scene generator.
Try in Studio
Image Classification
Image Classification
Label what the whole image shows using a trained classifier
Detects
Thousands of categories from the ImageNet dataset — animals, objects, textures, scenes, and more
Input
Single image or video frame; the model treats the entire frame as one scene
Output
A ranked list of category labels with confidence scores for the full image
Interaction
Showing different images or objects to the camera changes the system state; misclassification becomes a creative material
Failure cases
Ambiguous scenes, multiple competing subjects, images that fall outside ImageNet categories, low light
Project prompt
Found Image Fortune Teller
Show printed or found images to the camera and let the system misread them into generated messages, story fragments, or animations. Build a project where image recognition becomes a storytelling engine. The system should feel opinionated, poetic, or absurd rather than purely informative.
Try in Studio
Audio Classification
Audio Classification
Label ambient sounds from the microphone in real time
Detects
521 sound categories from the YAMNet model — speech, music, claps, taps, environmental noise, animal sounds, and more
Input
Microphone audio stream; no camera required
Output
A ranked list of sound category labels with confidence scores, updated several times per second
Interaction
Room sounds become triggers or continuous inputs; clapping, tapping, speaking, or ambient noise changes visuals or logic
Failure cases
Background noise interference, overlapping sounds, microphone quality differences across devices, silence producing low-confidence results
Project prompt
Noisy Room Translator
Make an interactive piece that listens to the room and reacts to its sound character. Ambient sounds such as claps, taps, voices, or room noise change visuals or game logic. Focus on atmosphere, rhythm, or misinterpretation rather than accuracy.
Try in Studio

Class Activity

Pick two tasks from this page. Try each one in MediaPipe Studio using your webcam or an uploaded file. For each task, write down:

  • What input it expects and what output it returns
  • One interaction idea it suggests to you
  • One limitation or surprise you noticed while testing it

Compare your two tasks. Which fits your current project idea better? Which would you want to build with if you had more time?
