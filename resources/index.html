Try before you code

MediaPipe Studio
MediaPipe Studio
A browser-based environment from Google where you can run every MediaPipe task — face landmark detection, gesture recognition, object detection, pose tracking, and more — directly in your browser using your webcam or uploaded media. No code required.
Use it for:
  • Testing whether a model detects what you need
  • Checking if tracking is stable enough for your idea
  • Adjusting confidence thresholds and seeing the effect
  • Exploring models before committing to building with them

Code examples

MediaPipe CodePen
MediaPipe CodePen Preview
The official MediaPipe preview profile on CodePen contains working examples of many MediaPipe tasks using the latest JavaScript API. Each pen shows the model in action and can be forked to experiment with directly in your browser.
Use it for:
  • Finding near-official examples close to the task APIs
  • Comparing multiple MediaPipe tasks side by side
  • Forking a working example as a starting point
  • Checking API patterns before writing from scratch

Creative communities

OpenProcessing
OpenProcessing
An online platform for running and sharing browser sketches built with Processing and p5.js. Every sketch.js in this project can be pasted into a new OpenProcessing sketch and run there without any local setup.
Use it for:
  • Running any sketch.js from this project without a local server
  • Browsing and remixing creative coding sketches
  • Sharing your work with others on the same platform
  • Sketch culture and community feedback

p5.js Gallery
p5.js Examples
The official p5.js examples and reference sketches cover the full range of what p5.js can do — drawing, interaction, typography, sound, and more. A practical reference for when you want to see how a particular behaviour is implemented.
Use it for:
  • Finding interaction and visual design patterns
  • Learning how to draw, animate, and respond to input in p5.js
  • Getting inspiration for how tracking data could drive visuals
  • Reference while building — not just browsing

Exploration Exercise

For each source above, find one example and answer the following questions. Write down your answers before opening any code.

  • What is the input? (webcam, image, audio, text, mouse, keyboard…)
  • What is the output? (visuals, sound, data, animation…)
  • What is the interaction rule that makes it interesting?
  • What is one thing you could change for your own project?
